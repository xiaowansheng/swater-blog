# 页面缓存机制

本文档合并原来的页面缓存说明、实现状态和快速参考。

## 1. 目标

页面缓存用于避免后台标签页切换时重复挂载组件，保留：

- 表单输入
- 滚动位置
- 页面筛选条件
- 已加载的数据和局部状态

## 2. 核心实现

当前方案基于自定义路由渲染和 `KeepAlive` 组件，不依赖 React Router 的默认 `<Outlet />` 缓存行为。

关键文件：

- [BasicLayout.tsx](../src/layout/BasicLayout.tsx)
- [RouteCache.tsx](../src/components/common/RouteCache.tsx)
- [tabs.ts](../src/store/tabs.ts)
- [TestCache.tsx](../src/pages/TestCache.tsx)

## 3. 如何配置

在 `routeConfig` 中配置 `keepAlive`：

```tsx
const routeConfig = [
  { path: '/dashboard', component: Dashboard, title: '仪表盘', keepAlive: true },
  { path: '/article/edit/:id', component: ArticleEdit, title: '编辑文章', keepAlive: true },
  { path: '/simple-page', component: SimplePage, title: '简单页面', keepAlive: false },
];
```

## 4. 适用场景

### 建议开启缓存

- 编辑页
- 列表页
- 配置页

### 建议关闭缓存

- 实时数据页
- 一次性操作页
- 简单静态页

## 5. 常用操作

### 刷新指定页面缓存

```ts
window.dispatchEvent(new CustomEvent('tab-refresh', {
  detail: { key: '/article/edit/123' }
}));
```

### 清理指定页面缓存

```ts
window.dispatchEvent(new CustomEvent('tab-remove', {
  detail: { key: '/article/edit/123' }
}));
```

## 6. 测试方法

1. 打开 `/test-cache`
2. 修改页面中的输入内容或计数器
3. 切换到其他标签页再切回来
4. 确认状态是否保留，组件是否没有重新挂载

## 7. 调试建议

- 查看 `BasicLayout` 和 `KeepAlive` 控制台日志
- 检查缓存页数量是否符合预期
- 关注事件监听器和副作用是否在页面失活时正确处理

## 8. 维护规则

- 页面缓存只保留一份主文档，即本文档
- 代码实现变化时，优先更新本文档和相关源码注释

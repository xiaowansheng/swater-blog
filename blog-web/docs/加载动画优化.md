# 加载动画优化

本文档合并原来的 `LOADING_*` 系列文档，作为唯一维护入口。

## 1. 目标

- 路由完成后尽快关闭加载动画
- 保留最小时长，避免闪烁
- 支持并发切页安全
- 对需要等待数据的页面提供 `PageReady` 机制

## 3. 默认配置

```ts
export const LOADING_CONFIG = {
  ROUTE_MIN_DURATION: 250,
  INITIAL_MIN_DURATION: 800,
  INITIAL_BRAND_ANIMATION: 3000,
  SHOW_THRESHOLD: 100,
  PROGRESS_INTERVAL: 100,
  PROGRESS_MAX_BEFORE_COMPLETE: 90,
};
```

## 4. 基础行为

### 路由切换

- 路由结束后立即进入关闭流程
- 至少显示 `ROUTE_MIN_DURATION`
- 使用 token 机制避免快速切页时状态错乱

### 首屏加载

- 首次访问保留品牌动画
- 最短 `INITIAL_MIN_DURATION`
- 最长 `INITIAL_BRAND_ANIMATION`

## 5. PageReady 用法

### 静态页面

```tsx
import { useAutoPageReady } from '@/lib/hooks/usePageReady';

export default function AboutPage() {
  useAutoPageReady();
  return <div>About</div>;
}
```

### 依赖数据的页面

```tsx
import { markPageReady } from '@/lib/hooks/usePageReady';

useEffect(() => {
  fetchData()
    .then(() => markPageReady())
    .catch(() => markPageReady());
}, []);
```

## 6. 测试清单

- 快速切换多个页面，确认不闪烁
- 清空 `sessionStorage` 后刷新，确认首屏动画正常
- 慢网环境下确认加载动画持续到真实完成
- 使用 `PageReady` 的页面确认不会过早关闭

## 7. 调优建议

- 高性能设备：适当调低 `ROUTE_MIN_DURATION`
- 低性能设备：适当提高 `INITIAL_MIN_DURATION`
- 除非页面确实依赖数据，不要滥用 `PageReady`

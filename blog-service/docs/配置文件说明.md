# 配置文件说明

本项目包含多个环境的配置文件，用于不同的部署和开发场景。

## 配置文件结构

```
src/main/resources/
├── application.yml              # 主配置文件
├── application-dev.yml          # 开发环境配置
├── application-test.yml         # 测试环境配置
├── application-docker.yml       # Docker环境配置
├── schema-dev.sql              # 开发环境数据库表结构
├── data-dev.sql                # 开发环境初始化数据
├── schema-test.sql             # 测试环境数据库表结构
└── data-test.sql               # 测试环境初始化数据
```

## 环境配置说明

### 1. 开发环境 (dev)

**配置文件**: `application-dev.yml`

**特点**:
- 使用本地MySQL数据库
- 启用详细的SQL日志输出
- 开启Spring DevTools热重载
- 暴露所有Actuator端点用于调试
- 禁用限流和IP黑名单
- 启用Swagger UI在线测试
- 支持跨域请求

**数据库配置**:
- 数据库: `blog_dev`
- 用户名: `root`
- 密码: `123456`
- 端口: `3306`

**启动方式**:
```bash
# 方式1: 通过配置文件指定
java -jar blog-service.jar --spring.profiles.active=dev

# 方式2: 通过环境变量
export SPRING_PROFILES_ACTIVE=dev
java -jar blog-service.jar

# 方式3: IDE中设置VM参数
-Dspring.profiles.active=dev
```

### 2. 测试环境 (test)

**配置文件**: `application-test.yml`

**特点**:
- 使用内存H2数据库，测试完成后自动清理
- 随机端口避免冲突
- 禁用外部服务依赖
- 启用详细的测试日志
- 简化的监控配置
- 支持测试数据自动初始化

**数据库配置**:
- 数据库: H2内存数据库
- 自动创建和删除表结构
- 预置测试数据

**使用场景**:
- 单元测试
- 集成测试
- CI/CD流水线

### 3. Docker环境 (docker)

**配置文件**: `application-docker.yml`

**特点**:
- 使用环境变量配置外部服务连接
- 优化的容器网络配置
- 完整的监控和日志配置
- 支持集群部署
- 生产级别的安全配置

**服务依赖**:
- MySQL: `mysql:3306`
- Redis: `redis:6379`
- RabbitMQ: `rabbitmq:5672`
- Elasticsearch: `elasticsearch:9200`

**启动方式**:
```bash
# 使用docker-compose
docker-compose up -d

# 单独启动
docker run -e SPRING_PROFILES_ACTIVE=docker blog-service
```

## 配置项说明

### 数据库配置

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/blog_dev
    username: root
    password: 123456
    hikari:
      maximum-pool-size: 20    # 最大连接数
      minimum-idle: 5          # 最小空闲连接数
      idle-timeout: 300000     # 空闲连接超时时间
```

### Redis配置

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      database: 0
      lettuce:
        pool:
          max-active: 8        # 最大连接数
          max-idle: 8          # 最大空闲连接数
```

### 监控配置

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  metrics:
    export:
      prometheus:
        enabled: true
```

### 安全配置

```yaml
security:
  rate-limit:
    enabled: true
    default-limit: 100       # 每分钟请求限制
  file-upload:
    max-size: 10MB          # 文件上传大小限制
    allowed-extensions:      # 允许的文件类型
      - jpg
      - png
      - pdf
```

## 环境切换

### 1. 通过配置文件

修改 `application.yml` 中的 `spring.profiles.active` 值：

```yaml
spring:
  profiles:
    active: dev  # 可选值: dev, test, docker
```

### 2. 通过启动参数

```bash
java -jar app.jar --spring.profiles.active=dev
```

### 3. 通过环境变量

```bash
export SPRING_PROFILES_ACTIVE=dev
java -jar app.jar
```

### 4. 通过IDE配置

在IDE的运行配置中设置VM参数：
```
-Dspring.profiles.active=dev
```

## 数据库初始化

### 开发环境

1. 创建数据库：
```sql
CREATE DATABASE blog_dev CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行初始化脚本：
```bash
mysql -u root -p blog_dev < src/main/resources/schema-dev.sql
mysql -u root -p blog_dev < src/main/resources/data-dev.sql
```

### 测试环境

测试环境使用H2内存数据库，会自动执行 `schema-test.sql` 和 `data-test.sql`。

## 常见问题

### 1. 数据库连接失败

检查数据库服务是否启动，用户名密码是否正确：

```bash
# 检查MySQL服务
systemctl status mysql

# 测试连接
mysql -u root -p -h localhost
```

### 2. Redis连接失败

检查Redis服务状态：

```bash
# 检查Redis服务
systemctl status redis

# 测试连接
redis-cli ping
```

### 3. 端口冲突

修改配置文件中的端口号：

```yaml
server:
  port: 8081  # 修改为其他端口
```

### 4. 文件上传路径问题

确保上传目录存在且有写权限：

```bash
mkdir -p uploads
chmod 755 uploads
```

## 最佳实践

1. **开发环境**: 使用 `dev` 配置，便于调试和开发
2. **测试环境**: 使用 `test` 配置，确保测试的独立性
3. **生产环境**: 使用 `docker` 配置，通过环境变量管理敏感信息
4. **配置管理**: 敏感信息不要直接写在配置文件中，使用环境变量
5. **日志级别**: 开发环境使用DEBUG，生产环境使用INFO或WARN
6. **监控配置**: 生产环境必须启用完整的监控和健康检查

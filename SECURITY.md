## 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请**不要**通过公开的 GitHub Issues 报告。

### 报告方式

请通过以下方式私下报告安全问题：

1. **邮件报告**（推荐）
   - 发送邮件至：[xiaowansheng@foxmail.com](mailto:xiaowansheng@foxmail.com)
   - 邮件主题：`[SECURITY] Swater Blog 安全漏洞报告`

2. **GitHub Security Advisory**
   - 访问仓库的 Security 标签页
   - 点击 "Report a vulnerability"

### 报告内容

请在报告中包含以下信息：

- 漏洞的详细描述
- 重现步骤
- 影响范围和严重程度
- 可能的修复建议
- 您的联系方式

### 响应时间

- **确认收到**：24 小时内
- **初步评估**：72 小时内
- **详细分析**：7 天内
- **修复发布**：根据严重程度，1-30 天

### 安全漏洞等级

| 等级 | 描述 | 响应时间 |
|------|------|----------|
| 🔴 严重 | 可导致系统完全妥协 | 24 小时 |
| 🟠 高危 | 可导致数据泄露或权限提升 | 72 小时 |
| 🟡 中危 | 可导致部分功能异常 | 7 天 |
| 🟢 低危 | 轻微安全问题 | 30 天 |

## 安全最佳实践

### 部署安全

1. **环境隔离**
   - 使用独立的生产环境
   - 限制网络访问
   - 定期更新系统补丁

2. **数据库安全**
   - 使用强密码
   - 启用 SSL 连接
   - 定期备份数据

3. **应用安全**
   - 定期更新依赖
   - 使用 HTTPS
   - 配置防火墙

### 配置安全

1. **敏感信息**
   - 使用环境变量存储密钥
   - 不要在代码中硬编码密码
   - 定期轮换密钥

2. **访问控制**
   - 实施最小权限原则
   - 定期审查用户权限
   - 启用多因素认证

3. **日志监控**
   - 记录安全相关事件
   - 监控异常访问
   - 设置告警机制

## 已知安全考虑

### 当前实现的安全措施

- ✅ SQL 注入防护（MyBatis-Plus）
- ✅ XSS 防护（输入验证）
- ✅ CSRF 防护（SaToken）
- ✅ 认证授权（SaToken）
- ✅ 接口限流（自定义实现）
- ✅ 输入验证（Hibernate Validator）
- ✅ 日志记录（操作日志）

### 需要注意的安全点

1. **文件上传**
   - 限制文件类型和大小
   - 扫描恶意文件
   - 隔离存储位置

2. **API 安全**
   - 实施接口限流
   - 验证请求来源
   - 记录访问日志

3. **数据安全**
   - 敏感数据加密
   - 定期数据备份
   - 访问权限控制

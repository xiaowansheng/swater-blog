# 后端配置参考

本文档合并原来的开发环境、配置说明和邮件配置文档，以 `blog-service/src/main/resources` 下的真实配置为准。

## 1. 生效文件

- [application.yml](../../blog-service/src/main/resources/application.yml)
- [application-dev.yml](../../blog-service/src/main/resources/application-dev.yml)
- [application-docker.yml](../../blog-service/src/main/resources/application-docker.yml)
- [schema.sql](../../blog-service/src/main/resources/schema.sql)
- [data-dev.sql](../../blog-service/src/main/resources/data-dev.sql)

说明：

- 当前仓库没有 `application-prod.yml`
- 当前仓库没有 `application-test.yml`
- 生产部署主要通过 `application-docker.yml + 环境变量`

## 2. Profile 选择

`application.yml` 只负责激活 profile：

```yaml
spring:
  profiles:
    active: ${SPRING_PROFILES_ACTIVE:dev}
```

常用值：

- `dev`
- `docker`

## 3. 开发环境重点配置

`application-dev.yml` 用于本机直接运行后端，例如：

```bash
./gradlew bootRun --args='--spring.profiles.active=dev'
```

根目录 [docker-compose.dev.yml](../../docker-compose.dev.yml) 虽然是开发编排，但其中的 `blog-service` 容器当前使用 `docker` profile。

### 3.1 数据库

- 默认库名：`blog`
- 默认用户：`blog`
- 默认密码使用 `change_me_*` 占位值
- 来源：`application-dev.yml` 的 `${DB_*:...}` 默认值和根目录 `.env.example`

### 3.2 Redis

- 默认地址：`localhost:6379`
- 密码默认留空

### 3.3 RabbitMQ

- 默认地址：`localhost:5672`
- 默认用户名：`admin`
- 默认密码使用 `change_me_*` 占位值

### 3.4 邮件

- 开发环境示例值仅作为占位符
- 不应直接把示例值用于真实部署
- 不再在文档中保留任何真实账号、授权码或密钥

### 3.5 Revalidate

- 开发环境默认：`http://127.0.0.1:3001`
- 对应本地 `blog-web` 开发端口

## 4. Docker 环境重点配置

Docker profile 由 Compose 命令固定，运行时依赖这些环境变量：

- `SERVER_PORT`
- `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USER` / `DB_PASSWORD`
- `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD`
- `RABBITMQ_HOST` / `RABBITMQ_USER` / `RABBITMQ_PASSWORD`

参考模板见 [.env.example](../../.env.example)。

`application-docker.yml` 还支持 `MAIL_*`、`JWT_SECRET_KEY`、`AUTH_EMAIL_SESSION_SECRET`、`FILE_STORAGE_*`、`OSS_*`、`QINIU_*`、`AMAP_KEY`、`BAIDU_AK`、`SEARCH_*`、`APP_CORS_*` 等变量。当前 [docker-compose.yml](../../docker-compose.yml) 默认没有把这些变量全部写入 `blog-service.environment`；需要使用时，应先补齐 Compose 环境变量或通过部署平台注入容器环境。

### 4.1 管理员初始化

管理员初始化配置由具体 profile YAML 提供，不再由根目录 `.env.example` 配置。

`application-dev.yml` 当前值：

```yaml
blog:
  bootstrap:
    admin:
      username: admin
      password: admin
      email: admin@example.com
```

`application-docker.yml` 当前值：

```yaml
blog:
  bootstrap:
    admin:
      username: admin
      password: change_me_admin_password
      email: admin@example.com
```

行为：

- 后端启动时默认执行管理员初始化检查
- 如果管理员用户不存在，会用上述配置创建账号和 `admin` 角色
- 如果管理员用户已存在，只跳过用户创建，不会重置密码
- `DataInitializer` 要求 `blog.bootstrap.admin.password` 非空；当前两个 profile 都提供了非空默认值
- 生产环境首次启动前应修改 `application-docker.yml` 中的 `blog.bootstrap.admin.*`，或先把该配置改为 `${...}` 形式再通过容器环境注入

## 5. 邮件配置

统一使用 `spring.mail.*`：

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: ${MAIL_USERNAME:}
    password: ${MAIL_PASSWORD:}
    from-name: ${MAIL_FROM_NAME:Swater Blog}
```

建议：

- 使用邮箱授权码，不使用登录密码
- 真实凭据只放在本机 `.env`、运行环境变量或部署平台密钥里；Docker 使用前需确认已注入容器环境
- 文档只保留字段含义，不保留真实值

## 6. 常用配置分组

### 6.1 基础设施

- `spring.datasource.*`
- `spring.data.redis.*`
- `spring.rabbitmq.*`
- `spring.mail.*`

### 6.2 业务安全

- `sa-token.*`
- `auth.crypto.*`
- `auth.email-session.*`

### 6.3 文件和位置服务

- `file.storage.*`
- `location.amap.*`
- `location.baidu.*`
- `location.ip2location.*`

### 6.4 观测与日志

- `logging.*`
- `management.*`
- `api-resource.auto-refresh-on-startup`

说明：

- `application-dev.yml` 设置 `management.server.port: 8081`，监控端点独立于主端口
- `application-docker.yml` 没有单独设置管理端口，Actuator 默认挂在 `SERVER_PORT` 上
- Compose 健康检查会先尝试 `8081`，再回退到 `$SERVER_PORT`，用于兼容这两类 profile

## 7. 本地联调建议

- 监控：`http://localhost:8081/actuator`
- 管理后台：`http://localhost:3000`
- 博客前台：`http://localhost:3001`

## 8. 维护规则

- 配置项变化时，优先更新实际 `application*.yml`
- 文档只解释“字段用途”和“默认行为”，不复制整份配置文件
- 敏感信息只能保留变量名，不允许保留真实凭据

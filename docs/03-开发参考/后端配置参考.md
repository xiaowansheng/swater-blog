## 2. Profile 选择

`application.yml` 只负责激活 profile：

```yaml
spring:
  profiles:
    active: ${SPRING_PROFILES_ACTIVE:dev}
```

常用值：

- `dev`
- `docker`

## 3. 开发环境重点配置

### 3.1 数据库

- 默认库名：`blog`
- 默认用户：`blog`
- 默认密码使用 `change_me_*` 占位值
- 来源：`application-dev.yml` 的 `${DB_*:...}` 默认值和根目录 `.env.example`

### 3.2 Redis

- 默认地址：`localhost:6379`
- 密码默认留空

### 3.3 RabbitMQ

- 默认地址：`localhost:5672`
- 默认用户名：`admin`
- 默认密码使用 `change_me_*` 占位值

### 3.4 邮件

- 开发环境示例值仅作为占位符
- 不应直接把示例值用于真实部署
- 不再在文档中保留任何真实账号、授权码或密钥

### 3.5 Revalidate

- 开发环境默认：`http://127.0.0.1:3001`
- 对应本地 `blog-web` 开发端口

## 4. Docker 环境重点配置

Docker profile 依赖环境变量：

- `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USER` / `DB_PASSWORD`
- `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD`
- `RABBITMQ_HOST` / `RABBITMQ_USER` / `RABBITMQ_PASSWORD`
- `MAIL_HOST` / `MAIL_PORT` / `MAIL_USERNAME` / `MAIL_PASSWORD`
- `JWT_SECRET_KEY`
- `AUTH_EMAIL_SESSION_SECRET`

参考模板见 [.env.example](/home/xiaowansheng/projects/develop-projects/swater-blog/.env.example)。

## 5. 邮件配置

统一使用 `spring.mail.*`：

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: ${MAIL_USERNAME:}
    password: ${MAIL_PASSWORD:}
    from-name: ${MAIL_FROM_NAME:Swater Blog}
```

建议：

- 使用邮箱授权码，不使用登录密码
- 真实凭据只放在 `.env` 或部署平台密钥里
- 文档只保留字段含义，不保留真实值

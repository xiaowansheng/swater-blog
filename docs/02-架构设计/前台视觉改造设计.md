# 前台视觉改造设计

本文档整合前台二次元视觉规范和实施路线图。

## 1. 设计目标

- 建立具有识别度的前台视觉语言
- 保持技术博客阅读体验，不让装饰压过内容
- 装饰与业务逻辑解耦，支持后续主题扩展

## 2. 风格关键词

- 清新
- 治愈
- 元气
- 轻量

主色调以薄荷绿为主，辅以樱花粉、星星黄、天空蓝。

## 3. 设计原则

- 克制：装饰不遮挡正文
- 语义化：颜色、动效和层级可通过 Token 管理
- 可关闭：特效必须可降级、可禁用
- 渐进增强：移动端和低性能设备优先保证可用性

## 4. 设计 Token

### 4.1 颜色

```css
:root {
  --background: #f0fdf4;
  --foreground: #14532d;
  --card: #ffffff;
  --card-border: #bbf7d0;
  --primary: #34d399;
  --deco-pink: #f9a8d4;
  --deco-yellow: #fde047;
  --deco-blue: #bae6fd;
}
```

### 4.2 动效分级

```css
:root {
  --motion-soft: 0.3;
  --motion-medium: 0.6;
  --motion-strong: 1;
}
```

## 5. 重点组件

- 全局背景：纯色底 + 轻纹理 + 可选粒子层
- 文章卡片：大圆角、淡边框、轻上浮
- 导航栏：毛玻璃与轻量 hover 反馈
- 看板娘：氛围组件，不承载主功能入口

## 6. 图层约束

- `z-0`：Canvas 背景
- `z-10`：静态纹理
- `z-20`：正文卡片
- `z-30`：导航与侧栏
- `z-40`：看板娘
- `z-50`：Modal / Toast

## 7. 分阶段落地

### 阶段一：基础设计语言

- 注入 CSS 变量
- 把语义 Token 映射到 Tailwind
- 调整字体和基础排版

### 阶段二：核心视觉重构

- 更新全局背景
- 重做文章卡片
- 调整导航和 Footer

### 阶段三：装饰系统

- `SakuraRain`
- `Mascot`
- `CornerDecoration`

### 阶段四：交互与反馈

- 页面进入、长阅读、滚动触底等触发器
- 点赞和切页微动效

### 阶段五：控制与性能

- 全局装饰开关
- 移动端降级策略
- 低性能设备特效降级

## 8. 实施建议

- 先落 Token，再做视觉替换，不要一边改组件一边改设计系统
- 新文档保留规范和阶段目标，具体组件实现放在模块文档中
- 实施记录应放在 PR 或变更说明，不再新增单独“路线图副本”

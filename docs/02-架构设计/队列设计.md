# 队列设计文档

## 1. 队列概述

使用RabbitMQ作为消息队列，实现异步通知处理。

## 2. 交换机设计

### 2.1 通知交换机 (notification.exchange)

- **类型**: Topic
- **用途**: 处理所有通知相关消息
- **持久化**: 是

## 3. 队列设计

### 3.1 评论通知队列

- **队列名**: `comment.notification.queue`
- **路由键**: `comment.notification`
- **用途**: 处理评论通知
- **持久化**: 是
- **消费者**: CommentNotificationConsumer

**消息格式**:
```json
{
  "type": "comment",
  "commentId": 123,
  "postId": 456,
  "commenterId": 789,
  "commenterName": "张三",
  "commentContent": "评论内容",
  "postTitle": "文章标题",
  "authorId": 111,
  "timestamp": "2024-01-01T00:00:00"
}
```

## 4. 路由规则

### 4.1 路由键规则

- `comment.notification`: 评论通知
- `login.notification`: 登录通知
- `system.notification`: 系统通知

### 4.2 绑定关系

```
notification.exchange (Topic)
├── comment.notification.queue (comment.notification)
├── login.notification.queue (login.notification)
└── system.notification.queue (system.notification)
```

## 5. 消息处理流程

### 5.1 评论通知流程

```
用户评论 
  → 保存评论到数据库
  → 发送MQ消息到comment.notification.queue
  → CommentNotificationConsumer消费消息
  → 查询文章作者
  → 检查通知配置
  → 发送WebSocket通知
  → 发送邮件通知（如果配置开启）
```

### 5.2 登录通知流程

```
用户登录
  → 验证成功
  → 发送MQ消息到login.notification.queue
  → LoginNotificationConsumer消费消息
  → 检查通知配置
  → 发送WebSocket通知
  → 发送邮件通知（如果配置开启）
```

## 7. 消息确认机制

### 7.1 手动确认

```java
@RabbitListener(queues = QueueConstant.COMMENT_NOTIFICATION_QUEUE)
public void handle(Message message, Channel channel) throws IOException {
    try {
        // 处理消息
        processMessage(message);
        // 手动确认
        channel.basicAck(message.getMessageProperties().getDeliveryTag(), false);
    } catch (Exception e) {
        // 拒绝消息，重新入队
        channel.basicNack(message.getMessageProperties().getDeliveryTag(), false, true);
    }
}
```

### 7.2 重试机制

- 最大重试次数: 3次
- 重试间隔: 递增（1s, 5s, 10s）
- 失败后: 记录到死信队列

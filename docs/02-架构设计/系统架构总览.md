# 系统架构总览

## 1. 项目组成

Swater Blog 由四个主要模块构成：

- `blog-service`：Spring Boot 后端，负责认证、内容、评论、统计、通知和文件服务
- `blog-admin`：React + Vite 管理后台
- `blog-web`：Next.js 博客前台
- `blog-tools/yuque-sync`：语雀与博客数据库同步工具

## 2. 整体架构

```text
blog-web / blog-admin / yuque-sync
                |
                v
           blog-service
                |
    +-----------+-----------+
    |           |           |
   MySQL      Redis      RabbitMQ
                |
          Elasticsearch（可选）
```

## 3. 后端分层

### 3.1 典型分层

- Controller：对外提供 `/api/public/*`、`/api/auth/*`、`/api/admin/*`
- Service：承载查询、写操作和业务编排
- Repository / Mapper：数据库访问
- Model：`entity` / `dto` / `vo`

### 3.2 横切能力

- Sa-Token：认证与授权
- AOP：操作日志、异常日志、限流
- RabbitMQ：评论通知、系统通知等异步任务
- Redis：配置、热点数据和用户态缓存
- Quartz：定时任务
- WebSocket：实时通知

## 4. 前后端职责

### blog-admin

- 内容管理
- 用户、角色、菜单、资源管理
- 系统配置、日志、访客统计

### blog-web

- 文章阅读与互动
- 搜索、评论、留言、友链
- SEO、国际化、主题和视觉体验

### yuque-sync

- 导入、导出、双向同步
- 映射关系与日志查看
- 独立 SQLite 元数据存储

## 5. 关键业务链路

### 5.1 文章发布

```text
管理后台 -> blog-service -> MySQL
                       -> 缓存失效
                       -> 可选搜索索引更新
```

### 5.2 评论通知

```text
博客前台 -> blog-service -> MySQL
                       -> RabbitMQ
                       -> 通知消费者
                       -> WebSocket / 邮件
```

### 5.3 统计追踪

```text
博客前台埋点 -> blog-service -> visitor / session / page_view / content_read
```

- 单一事实源：端口、脚本、环境变量以仓库配置文件为准
- 模块就近：强耦合的专项文档保留在模块目录

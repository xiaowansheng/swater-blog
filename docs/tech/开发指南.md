# 开发指南

## 1. 开发环境搭建

### 1.1 安装JDK 21

```bash
# 下载JDK 21
# 配置JAVA_HOME环境变量
export JAVA_HOME=/path/to/jdk-21
export PATH=$JAVA_HOME/bin:$PATH
```

### 1.2 安装Gradle

```bash
# 下载Gradle 8.0+
# 配置GRADLE_HOME环境变量
export GRADLE_HOME=/path/to/gradle
export PATH=$GRADLE_HOME/bin:$PATH
```

### 1.3 安装IDE

推荐使用 IntelliJ IDEA 2023.1+，配置：
- JDK 21
- Gradle
- Lombok插件
- MyBatis插件

### 1.4 安装数据库和中间件

使用Docker快速启动：
```bash
docker-compose up -d mysql redis rabbitmq elasticsearch
```

## 2. 项目结构说明

### 2.1 包结构

```
com.blog
├── config/          # 配置类
├── controller/      # 控制器
│   ├── public/      # 公开接口
│   └── admin/       # 管理接口
├── service/         # 服务层
├── mapper/          # MyBatis Mapper
├── model/           # 模型层
│   ├── entity/      # 实体类
│   ├── dto/         # 数据传输对象
│   └── vo/          # 视图对象
├── annotation/      # 自定义注解
├── aspect/          # AOP切面
├── util/            # 工具类
├── listener/        # 监听器
├── exception/       # 异常处理
└── constant/        # 常量
```

### 2.2 命名规范

- **类名**: 大驼峰（PascalCase）
- **方法名/变量名**: 小驼峰（camelCase）
- **常量**: 大写下划线（UPPER_SNAKE_CASE）
- **包名**: 小写（lowercase）

## 3. 开发规范

### 3.1 代码规范

- 使用4个空格缩进
- 每行代码不超过120字符
- 类和方法添加JavaDoc注释
- 使用Lombok简化代码

### 3.2 实体类规范

```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("post")
public class Post extends BaseEntity {
    @TableId(type = IdType.AUTO)
    private Long id;
    
    private String title;
    
    // 逻辑删除字段在BaseEntity中
}
```

### 3.3 Service规范

```java
// 查询接口
public interface PostPublicQueryService {
    PostVO getById(Long id);
    PageResult<PostVO> list(PostQueryDTO dto);
}

// 操作接口
public interface PostAdminCommandService {
    Long create(PostDTO dto);
    void update(Long id, PostDTO dto);
    void delete(Long id);
}

// 实现类
@Service
public class PostServiceImpl implements PostPublicQueryService, 
                                       PostAdminQueryService, 
                                       PostAdminCommandService {
    // 实现方法
}
```

### 3.4 Controller规范

```java
@RestController
@RequestMapping("/api/public/post")
@Api(tags = "文章公开接口")
public class PostPublicController {
    
    @Autowired
    private PostPublicQueryService queryService;
    
    @GetMapping("/{id}")
    @ApiOperation("获取文章详情")
    public Result<PostVO> getById(@PathVariable Long id) {
        return Result.success(queryService.getById(id));
    }
}
```

### 3.5 Mapper规范

```java
@Mapper
public interface PostMapper extends BaseMapper<Post> {
    // 自定义查询方法
    Page<Post> selectPageWithCategory(Page<Post> page, @Param("categoryId") Long categoryId);
}
```

### 4.6 ES搜索

```java
@Autowired
private ElasticsearchRestTemplate esTemplate;

public Page<PostDocument> search(String keyword, Pageable pageable) {
    NativeSearchQueryBuilder queryBuilder = new NativeSearchQueryBuilder();
    queryBuilder.withQuery(QueryBuilders.multiMatchQuery(keyword, "title", "content"));
    queryBuilder.withPageable(pageable);
    return esTemplate.search(queryBuilder.build(), PostDocument.class);
}
```

## 7. 常见问题

### 7.1 MyBatis-Plus逻辑删除不生效

检查配置：
```yaml
mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

### 7.2 时间字段自动填充不生效

检查MetaObjectHandler配置：
```java
@Component
public class MyMetaObjectHandler implements MetaObjectHandler {
    @Override
    public void insertFill(MetaObject metaObject) {
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
}
```

### 7.3 SaToken权限校验不生效

检查配置和注解：
```java
@SaCheckRole("admin")  // 角色校验
@SaCheckPermission("post:create")  // 权限校验
```

### 7.4 Redis缓存不生效

检查Redis连接和配置：
```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

## 8. 代码提交规范

### 8.1 Commit Message格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

示例：
```
feat(post): 添加文章置顶功能

- 添加isTop字段
- 实现置顶逻辑
- 更新相关接口
```

### 8.2 分支管理

- `main`: 主分支，生产环境
- `develop`: 开发分支
- `feature/*`: 功能分支
- `bugfix/*`: 修复分支
- `hotfix/*`: 热修复分支

## 9. 性能优化建议

### 9.1 数据库查询优化

- 使用索引
- 避免N+1查询
- 使用分页
- 合理使用缓存

### 9.2 代码优化

- 避免循环中查询数据库
- 使用批量操作
- 异步处理耗时操作
- 合理使用缓存

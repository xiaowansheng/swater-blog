# 部署文档

## 1. 环境要求

### 1.1 服务器要求
- **操作系统**: Linux (推荐 Ubuntu 20.04+ 或 CentOS 7+)
- **CPU**: 2核+
- **内存**: 4GB+
- **磁盘**: 20GB+

### 1.2 软件要求
- **JDK**: 21+
- **MySQL**: 8.0+
- **Redis**: 6.0+
- **RabbitMQ**: 3.12+
- **Elasticsearch**: 8.0+
- **Docker**: 20.10+ (可选)
- **Docker Compose**: 2.0+ (可选)

## 2. 传统部署方式

### 2.1 数据库初始化

```bash
# 创建数据库
mysql -u root -p
CREATE DATABASE blog DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入SQL脚本（如果有）
mysql -u root -p blog < init.sql
```

### 2.2 安装中间件

#### 2.2.1 安装Redis
```bash
# Ubuntu
sudo apt update
sudo apt install redis-server

# CentOS
sudo yum install redis
sudo systemctl start redis
sudo systemctl enable redis
```

#### 2.2.2 安装RabbitMQ
```bash
# 安装Erlang
wget https://packages.erlang-solutions.com/erlang-solutions_2.0_all.deb
sudo dpkg -i erlang-solutions_2.0_all.deb
sudo apt-get update
sudo apt-get install erlang

# 安装RabbitMQ
sudo apt-get install rabbitmq-server
sudo systemctl start rabbitmq-server
sudo systemctl enable rabbitmq-server

# 创建用户
sudo rabbitmqctl add_user admin password
sudo rabbitmqctl set_user_tags admin administrator
sudo rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"
```

#### 2.2.3 安装Elasticsearch
```bash
# 下载并安装
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.11.0-linux-x86_64.tar.gz
tar -xzf elasticsearch-8.11.0-linux-x86_64.tar.gz
cd elasticsearch-8.11.0

# 配置
vim config/elasticsearch.yml
# 设置 network.host: 0.0.0.0

# 启动
./bin/elasticsearch -d
```

### 2.3 构建应用

```bash
# 克隆项目
git clone <repository-url>
cd swater-blog/blog

# 构建
./gradlew clean build -x test

# 生成jar包
./gradlew bootJar
```

### 2.4 配置应用

```bash
# 复制配置文件
cp src/main/resources/application-prod.yml src/main/resources/application-prod.yml.bak

# 编辑配置文件
vim src/main/resources/application-prod.yml
# 配置数据库、Redis、RabbitMQ、ES等连接信息
```

### 2.5 启动应用

```bash
# 方式1: 直接启动
java -jar build/libs/blog-1.0.0.jar

# 方式2: 后台启动
nohup java -jar build/libs/blog-1.0.0.jar > blog.log 2>&1 &

# 方式3: 使用systemd
sudo vim /etc/systemd/system/blog.service
```

systemd服务文件示例：
```ini
[Unit]
Description=Swater Blog Application
After=network.target

[Service]
Type=simple
User=blog
WorkingDirectory=/opt/blog
ExecStart=/usr/bin/java -jar /opt/blog/blog-1.0.0.jar
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start blog
sudo systemctl enable blog
```

## 3. Docker部署方式

### 3.1 使用Docker Compose

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: blog-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: blog
      MYSQL_CHARACTER_SET_SERVER: utf8mb4
      MYSQL_COLLATION_SERVER: utf8mb4_unicode_ci
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - blog-network

  redis:
    image: redis:7-alpine
    container_name: blog-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - blog-network

  rabbitmq:
    image: rabbitmq:3-management-alpine
    container_name: blog-rabbitmq
    environment:
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123
    ports:
      - "5672:5672"
      - "15672:15672"
    volumes:
      - rabbitmq_data:/var/lib/rabbitmq
    networks:
      - blog-network

  elasticsearch:
    image: elasticsearch:8.11.0
    container_name: blog-elasticsearch
    environment:
      - discovery.type=single-node
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - xpack.security.enabled=false
    ports:
      - "9200:9200"
    volumes:
      - es_data:/usr/share/elasticsearch/data
    networks:
      - blog-network

  blog:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: blog-app
    ports:
      - "8888:8888"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_NAME=blog
      - DB_USER=root
      - DB_PASSWORD=root123
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - RABBITMQ_HOST=rabbitmq
      - RABBITMQ_PORT=5672
      - RABBITMQ_USER=admin
      - RABBITMQ_PASSWORD=admin123
      - ES_URIS=http://elasticsearch:9200
    depends_on:
      - mysql
      - redis
      - rabbitmq
      - elasticsearch
    networks:
      - blog-network
    restart: always

volumes:
  mysql_data:
  redis_data:
  rabbitmq_data:
  es_data:

networks:
  blog-network:
    driver: bridge
```

创建 `Dockerfile`:

```dockerfile
FROM openjdk:21-jdk-slim

WORKDIR /app

COPY build/libs/*.jar app.jar

EXPOSE 8888

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 3.1.1 本地编译产物 + Docker Compose（可选）

如果希望在本地编译后再构建镜像（避免容器内编译），可以通过 `.env` 选择本地构建 Dockerfile：

```bash
# 后端
cd blog-service
./gradlew bootJar

# 后台管理
cd blog-admin
pnpm build

# 前台
cd blog-web
pnpm build
```

在项目根目录 `.env` 中设置：

```bash
BLOG_SERVICE_DOCKERFILE=Dockerfile.local
BLOG_ADMIN_DOCKERFILE=Dockerfile.local
BLOG_WEB_DOCKERFILE=Dockerfile.local
```

然后重新构建并启动：

```bash
docker-compose build blog-service blog-admin blog-web
docker-compose up -d
```

### 3.2 启动Docker服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f blog

# 停止服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v
```

## 4. CI/CD配置

### 4.1 GitHub Actions

创建 `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 21
      uses: actions/setup-java@v3
      with:
        java-version: '21'
        distribution: 'temurin'
    
    - name: Grant execute permission for gradlew
      run: chmod +x gradlew
    
    - name: Build with Gradle
      run: ./gradlew clean build -x test
    
    - name: Upload build artifacts
      uses: actions/upload-artifact@v3
      with:
        name: build-artifacts
        path: build/libs/*.jar
```

创建 `.github/workflows/cd.yml`:

```yaml
name: CD

on:
  push:
    branches: [ main ]
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 21
      uses: actions/setup-java@v3
      with:
        java-version: '21'
        distribution: 'temurin'
    
    - name: Build with Gradle
      run: ./gradlew clean build -x test
    
    - name: Build Docker image
      run: |
        docker build -t blog:${{ github.sha }} .
        docker tag blog:${{ github.sha }} blog:latest
    
    - name: Deploy to server
      uses: appleboy/ssh-action@master
      with:
        host: ${{ secrets.HOST }}
        username: ${{ secrets.USERNAME }}
        key: ${{ secrets.SSH_KEY }}
        script: |
          cd /opt/blog
          docker-compose pull
          docker-compose up -d --build
          docker system prune -f
```

### 4.2 配置Secrets

在GitHub仓库设置中添加：
- `HOST`: 服务器地址
- `USERNAME`: SSH用户名
- `SSH_KEY`: SSH私钥

## 5. Nginx反向代理

### 5.1 安装Nginx

```bash
# Ubuntu
sudo apt install nginx

# CentOS
sudo yum install nginx
```

### 5.2 配置Nginx

创建 `/etc/nginx/sites-available/blog`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket支持
    location /ws {
        proxy_pass http://localhost:8888;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态文件
    location /uploads {
        alias /opt/blog/uploads;
        expires 30d;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/blog /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 10. 故障排查

### 10.1 应用无法启动

```bash
# 检查端口占用
netstat -tlnp | grep 8888

# 检查日志
tail -f logs/blog.log

# 检查Java进程
ps aux | grep java
```

### 10.2 数据库连接失败

```bash
# 测试连接
mysql -h localhost -u root -p

# 检查防火墙
sudo ufw status
```

### 10.3 Redis连接失败

```bash
# 测试连接
redis-cli ping

# 检查配置
redis-cli CONFIG GET "*"
```

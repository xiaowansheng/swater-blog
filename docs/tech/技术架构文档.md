## 2. 架构设计

### 2.1 分层架构

```
┌─────────────────────────────────────┐
│         Controller Layer            │
│  (public/admin 接口控制器)           │
└─────────────────────────────────────┘
                 │
┌─────────────────────────────────────┐
│          Service Layer               │
│  (PublicQuery/AdminQuery/Command)    │
└─────────────────────────────────────┘
                 │
┌─────────────────────────────────────┐
│        Repository Layer              │
│  (Query/Command Repository)          │
└─────────────────────────────────────┘
                 │
┌─────────────────────────────────────┐
│          Model Layer                 │
│  (Entity/DTO/VO)                     │
└─────────────────────────────────────┘
```

### 2.2 技术选型说明

#### 2.2.1 MyBatis-Plus
- **优势**: 
  - 自动CRUD，减少代码量
  - 内置分页插件
  - 逻辑删除支持
  - 自动填充时间字段
- **使用场景**: 所有数据库操作

#### 2.2.2 SaToken
- **优势**:
  - 轻量级，API简单
  - 支持多种认证方式
  - 权限注解支持
- **使用场景**: 用户认证、角色权限、接口权限

#### 2.2.3 RabbitMQ
- **使用场景**:
  - 评论通知异步处理
  - 登录通知异步处理
  - 系统通知异步处理
- **队列设计**: 见队列设计文档

#### 2.2.4 Elasticsearch
- **使用场景**:
  - 文章全文搜索
  - 说说搜索
  - 评论搜索
- **索引设计**: 见ES索引设计

#### 2.2.5 Redis
- **使用场景**:
  - 配置信息缓存
  - 热点文章缓存
  - 用户信息缓存
  - 登录Token存储

#### 2.2.6 Quartz
- **使用场景**:
  - 日志清理任务（每天执行）
  - 统计数据任务（每小时执行）
  - 归档生成任务（每天执行）

#### 2.2.7 WebSocket
- **使用场景**:
  - 实时通知推送
  - 评论实时更新
  - 系统消息推送

## 3. 核心设计模式

### 3.1 接口分离原则
- **PublicQueryService**: 公开查询接口
- **AdminQueryService**: 管理查询接口
- **AdminCommandService**: 管理操作接口

### 3.2 Repository分离
- **QueryRepository**: 只读查询操作
- **CommandRepository**: 写操作

### 3.3 注解驱动
- **@ApiResource**: 接口资源自动注册
- **@LogOperation**: 操作日志自动记录
- **@RateLimit**: 接口限流

### 3.4 AOP切面
- **日志切面**: 自动记录操作日志和异常日志
- **权限切面**: 自动校验接口权限
- **限流切面**: 自动限流控制

## 4. 数据流设计

### 4.1 文章发布流程
```
Controller → Service → Repository → MySQL
                ↓
           更新ES索引
                ↓
           清除Redis缓存
```

### 4.2 评论流程
```
Controller → Service → Repository → MySQL
                ↓
           发送MQ消息 → 异步处理
                ↓
         WebSocket推送 + 邮件通知
```

### 4.3 搜索流程
```
Controller → SearchService → Elasticsearch
                ↓
            返回结果
```

### 4.4 缓存流程
```
Controller → Service → 检查Redis缓存
                ↓
        缓存未命中 → 查询数据库 → 写入缓存
```

## 5. 安全设计

### 5.1 认证流程
```
用户登录 → 验证用户名密码 → 生成Token → 存储Redis → 返回Token
```

### 5.2 授权流程
```
请求 → 验证Token → 获取用户角色 → 检查接口权限 → 允许/拒绝
```

### 5.3 数据安全
- 密码BCrypt加密
- SQL注入防护（MyBatis-Plus参数化查询）
- XSS防护（输入过滤）
- CSRF防护（Token验证）

## 6. 性能优化

### 6.1 缓存策略
- **配置缓存**: 1小时过期，启动时加载
- **文章缓存**: 30分钟过期，更新时清除
- **用户缓存**: 2小时过期，登出时清除

### 6.2 异步处理
- 邮件通知异步（RabbitMQ）
- 日志记录异步（操作日志同步记录耗时）
- 统计任务定时执行（Quartz）

### 6.3 数据库优化
- 合理索引设计
- 分页查询
- 逻辑删除（避免物理删除）
- 连接池优化

### 6.4 ES优化
- 索引分片设置
- 搜索高亮优化
- 搜索结果缓存

## 7. 扩展性设计

### 7.1 存储扩展
- 文件存储支持本地和OSS切换
- 配置化切换，无需修改代码

### 7.2 搜索扩展
- ES索引可扩展
- 支持多索引搜索

### 7.3 通知扩展
- 支持多种通知方式
- 可配置通知模板
- 支持自定义通知渠道

### 8.2 异常日志

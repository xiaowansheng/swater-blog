# 接口设计文档

## 1. 接口规范

### 1.1 接口分类
- **公开接口**: `/api/public/*` - 无需认证，用户端访问
- **管理接口**: `/api/admin/*` - 需要认证，管理端访问

### 1.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2024-01-01T00:00:00"
}
```

### 1.3 HTTP状态码
- `200`: 成功
- `400`: 请求参数错误
- `401`: 未认证
- `403`: 无权限
- `404`: 资源不存在
- `500`: 服务器错误

## 2. 公开接口

### 2.1 文章接口

#### 获取文章列表
```
GET /api/public/post/list
参数:
  - page: 页码（默认1）
  - size: 每页数量（默认10）
  - categoryId: 分类ID（可选）
  - tagId: 标签ID（可选）
  - keyword: 关键词（可选）
响应: PageResult<PostVO>
```

#### 获取文章详情
```
GET /api/public/post/{id}
响应: PostVO
```

#### 获取热门文章
```
GET /api/public/post/hot
参数:
  - limit: 数量（默认10）
响应: List<PostVO>
```

#### 获取最新文章
```
GET /api/public/post/latest
参数:
  - limit: 数量（默认10）
响应: List<PostVO>
```

## 4. 认证接口

### 4.1 登录
```
POST /api/auth/login
请求体: {username: string, password: string}
响应: {token: string, user: UserVO}
```

### 4.2 登出
```
POST /api/auth/logout
响应: void
```

### 4.3 获取当前用户信息
```
GET /api/auth/current
响应: UserVO
```

### 4.4 刷新Token
```
POST /api/auth/refresh
响应: {token: string}
```

# 数据库设计文档

## 1. 数据库概述

- **数据库类型**: MySQL 8.0+
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **逻辑删除**: 所有业务表使用 `deleted` 字段（0-未删除，1-已删除）
- **时间字段**: 所有业务表包含 `create_time` 和 `update_time`
- **ID类型**: 统一使用 `BIGINT` 类型

## 5. 表命名规范

- ✅ 去掉不必要的 `t_` 前缀
- ✅ 系统表使用 `sys_` 前缀（sys_api, sys_config, sys_notification, sys_menu）
- ✅ 日志表使用 `log_` 前缀（log_operation, log_error）
- ✅ 关联表使用清晰的后缀（article_tag, role_api, role_menu）
- ✅ 业务表使用单数形式（user, role, article, comment等）

## 6. 索引设计原则

- 主键自动创建聚簇索引
- 外键字段创建索引
- 查询频繁字段创建索引
- 唯一约束字段创建唯一索引
- 组合查询创建联合索引

## 7. 外键约束说明

- 用户相关：`user` 表作为主表，其他表通过 `user_id` 关联
- 文章相关：`article` 表通过 `author_id` 关联 `user`，通过 `category_id` 关联 `category`
- 评论相关：`comment` 表可关联 `article` 或 `talk`
- 文件相关：`file_reference` 表关联 `file_meta`
- 访客相关：`visitor_session` / `page_view` 表关联 `visitor`

所有外键约束使用 `ON DELETE CASCADE` 或 `ON DELETE SET NULL` 策略，确保数据一致性。

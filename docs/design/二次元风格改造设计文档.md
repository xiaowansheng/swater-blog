# 📋 swater-blog 前台二次元风格改造设计规范（完整版）

> 本文档用于指导 **swater-blog 前台 UI 的二次元风格重构与长期演进**，目标是在保证可读性、性能与可维护性的前提下，构建一个 **清新治愈、克制不浮夸、可持续扩展** 的二次元博客前台。

---

## 一、设计目标与原则

### 1.1 核心目标

* 🎨 打造具有明确识别度的二次元博客视觉风格
* 📖 不牺牲技术博客的阅读体验
* 🧩 装饰系统与业务逻辑彻底解耦
* 🛠 支持后续主题扩展（季节 / 节日 / 暗色）

### 1.2 设计原则

| 原则   | 说明            |
| ---- | ------------- |
| 克制   | 装饰永远不压过内容     |
| 语义化  | 所有设计变量可被“人理解” |
| 可关闭  | 所有特效均可被关闭     |
| 渐进增强 | 老设备 / 移动端体验优先 |

---

## 二、整体风格定义

* **关键词**：清新、治愈、元气、轻量
* **主色调**：薄荷绿（Mint / Emerald 系）
* **辅助色**：樱花粉、星星黄、天空蓝
* **风格参照**：日系博客 / Galgame UI / 二次元轻社区

## 四、动画系统规范

### 4.1 动画强度分级

```css
:root {
  --motion-soft: 0.3;
  --motion-medium: 0.6;
  --motion-strong: 1;
}
```

| 等级     | 使用场景          |
| ------ | ------------- |
| soft   | 呼吸、悬浮         |
| medium | hover / focus |
| strong | 点击、点赞、反馈      |

---

### 4.2 核心动画定义

#### 🌸 悬浮呼吸

```css
@keyframes float {
  0%,100% { transform: translateY(0); }
  50% { transform: translateY(-10px); }
}
.animate-float {
  animation: float 6s ease-in-out infinite;
}
```

#### ✨ 闪烁动画

```css
@keyframes twinkle {
  0%,100% { opacity: 1; transform: scale(1); }
  50% { opacity: 0.5; transform: scale(0.8); }
}
```

## 五、核心组件设计

### 5.1 全局背景系统

**层级结构**：

1. Base：纯色背景
2. Pattern：radial-gradient 波点
3. Effect：Canvas 樱花 / 粒子

* 粒子密度：低
* 禁止覆盖正文区域
* 支持 reduced-motion

---

### 5.2 看板娘（Mascot）系统

#### 功能定位

> 看板娘是“氛围引导角色”，不是功能入口。

#### 行为触发模型

```ts
type MascotEvent =
  | 'page_enter'
  | 'long_read'
  | 'scroll_bottom'
  | 'theme_switch';
```

#### 交互行为示例

| 事件         | 文案示例               |
| ---------- | ------------------ |
| page_enter | 欢迎来到 swater-blog ✨ |
| long_read  | 要不要看看相关文章？         |
| night      | 晚上也要注意休息哦 🌙       |

---

### 5.3 文章卡片（Article Card）

* 圆角：`rounded-3xl`
* 边框：淡绿描边
* Hover：上浮 + 薄荷绿发光
* 分类角标：丝带 / 书签样式

---

### 5.4 标签与图标

* 标签：胶囊形 + 马卡龙色
* 图标库：Phosphor Icons（Fill / Duotone）

## 六、装饰组件清单

| 组件                     | 职责     |
| ---------------------- | ------ |
| `<SakuraRain />`       | 全局背景粒子 |
| `<Mascot />`           | 看板娘系统  |

所有组件：

* 不依赖业务状态
* 可配置开关

---

## 七、装饰控制系统（强烈推荐）

```ts
interface DecorationConfig {
  sakura: boolean;
  particles: boolean;
  mascot: boolean;
}
```

UI 提供：

* 装饰等级：关 / 少量 / 完整
* 移动端默认：少量

---

## 八、内容区设计红线（Anti-Pattern）

🚫 禁止事项：

* 正文段落中出现闪烁动画
* 粒子覆盖正文内容
* 同屏超过 2 种动态装饰
* 高饱和纯色大面积背景
* 移动端启用全屏粒子

📌 内容区优先级永远最高。

> **二次元风格不是“花”，而是“温度”。**
> 本设计以“长期使用 + 可演进”为目标，避免一次性炫技，实现一个真正耐看的二次元博客前台。

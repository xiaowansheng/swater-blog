# 博客前台设计文档

## 1. 系统概述

### 1.1 系统定位
Blog Web 是 Swater Blog 的博客前台系统，面向访客提供文章浏览、搜索、评论互动等功能，注重SEO优化和用户体验。

### 1.2 系统目标
- 优秀的SEO表现，提升搜索引擎收录
- 快速的首屏加载和流畅的交互体验
- 响应式设计，完美适配多端设备
- 现代化的UI设计，提供良好的阅读体验
- 支持实时评论和互动功能

## 3. 渲染策略

### 3.1 ISR (增量静态再生)
适用于内容相对静态、需要SEO的页面：
- **文章详情页**: `revalidate: 3600` (1小时)
- **首页**: `revalidate: 300` (5分钟)
- **分类页**: `revalidate: 600` (10分钟)
- **标签页**: `revalidate: 600` (10分钟)
- **归档页**: `revalidate: 3600` (1小时)
- **关于页**: `revalidate: 3600` (1小时)
- **友链页**: `revalidate: 1800` (30分钟)

### 3.2 SSR (服务端渲染)
适用于需要实时数据的页面：
- **搜索页**: 根据用户输入实时搜索
- **评论列表**: 实时加载最新评论
- **留言板**: 实时提交和展示

### 3.3 CSR (客户端渲染)
适用于纯交互功能：
- **评论提交**: 客户端提交后更新
- **浏览量统计**: 客户端异步更新
- **点赞功能**: 客户端交互

### 3.4 混合策略
- 文章详情页：ISR预渲染 + 客户端更新浏览量/评论数
- 首页：ISR预渲染 + 客户端加载最新评论
- 搜索页：SSR实时搜索 + 客户端缓存结果

### 7.3 请求封装

#### 服务端请求
```typescript
// lib/api/server.ts
export async function fetchServer<T>(url: string): Promise<T> {
  const baseUrl = process.env.API_BASE_URL || 'http://localhost:8888'
  const response = await fetch(`${baseUrl}${url}`, {
    cache: 'no-store' // SSR
    // 或 revalidate: 3600 // ISR
  })
  const data = await response.json()
  return data.data
}
```

#### 客户端请求
```typescript
// lib/api/client.ts
export async function fetchClient<T>(url: string): Promise<T> {
  const response = await fetch(url)
  const data = await response.json()
  return data.data
}
```

# 系统设计文档

## 1. 系统概述

### 1.1 系统定位
Swater Blog 是一个功能完整的博客系统，支持文章发布、评论互动、访客统计、实时通知等核心功能。

### 1.2 系统目标
- 提供高性能的博客内容管理
- 支持灵活的权限控制
- 完整的操作日志追踪
- 丰富的统计和可视化功能

## 2. 系统架构

### 2.1 整体架构

```
┌─────────────┐     ┌─────────────┐
│  blog-web   │     │ blog-admin  │
│   (前端)    │     │  (管理后台)  │
└──────┬──────┘     └──────┬──────┘
       │                   │
       └─────────┬─────────┘
                 │
         ┌───────▼───────┐
         │  blog (后端)   │
         └───────┬───────┘
                 │
    ┌────────────┼────────────┐
    │            │            │
┌───▼───┐  ┌───▼───┐  ┌───▼───┐
│ MySQL │  │ Redis │  │RabbitMQ│
└───────┘  └───────┘  └───────┘
    │
┌───▼───────┐
│Elasticsearch│
└────────────┘
```

### 2.2 分层架构

```
Controller Layer (控制器层)
    ├── public/     # 公开接口
    └── admin/      # 管理接口

Service Layer (服务层)
    ├── PublicQueryService    # 公开查询服务
    ├── AdminQueryService     # 管理查询服务
    └── AdminCommandService   # 管理操作服务

Repository Layer (数据访问层)
    ├── query/      # 查询Repository
    └── command/    # 操作Repository

Model Layer (模型层)
    ├── entity/     # 实体类
    ├── dto/        # 数据传输对象
    └── vo/         # 视图对象
```

## 3. 核心模块设计

### 3.1 用户模块
- **功能**: 用户注册、登录、权限管理
- **特点**: 基于SaToken的认证授权，支持角色和接口资源权限

### 3.2 文章模块
- **功能**: 文章CRUD、分类管理、标签管理、归档
- **特点**: 支持Markdown、草稿、发布状态、置顶

### 3.3 说说模块
- **功能**: 富文本说说发布、点赞、评论
- **特点**: 记录IP、地理位置、设备信息

### 3.4 评论模块
- **功能**: 评论、回复、审核
- **特点**: 支持文章评论和说说评论，记录完整访问信息

### 3.5 访客模块
- **功能**: 访客记录、统计分析
- **特点**: IP定位、地理位置、设备信息、访问统计

### 3.6 日志模块
- **功能**: 操作日志、异常日志
- **特点**: 记录操作详情、耗时、IP、参数等

### 3.7 配置模块
- **功能**: 系统配置管理
- **特点**: 支持网站信息、作者信息、通知设置、隐私设置等

### 3.8 文件模块
- **功能**: 文件上传、管理
- **特点**: 支持本地存储和OSS，图片压缩，类型限制

### 3.9 通知模块
- **功能**: 实时通知、邮件通知
- **特点**: WebSocket实时推送，RabbitMQ异步处理，HTML邮件模板

### 3.10 搜索模块
- **功能**: 全文搜索
- **特点**: 基于Elasticsearch，支持文章、说说、评论搜索

## 4. 接口设计原则

### 4.1 接口分类
- **公开接口** (`/api/public/*`): 无需认证，用户端访问
- **管理接口** (`/api/admin/*`): 需要认证，管理端访问

### 4.2 接口规范
- RESTful风格
- 统一响应格式
- 统一异常处理
- 接口资源注解自动注册

## 5. 数据流设计

### 5.1 文章发布流程
```
用户提交 → 数据校验 → 保存数据库 → 更新ES索引 → 清除缓存 → 返回结果
```

### 5.2 评论流程
```
用户评论 → 数据校验 → 保存数据库 → 发送MQ消息 → 异步处理通知 → WebSocket推送 → 邮件通知
```

### 5.3 搜索流程
```
用户搜索 → 查询ES → 返回结果 → 记录搜索日志
```

## 6. 安全设计

### 6.1 认证授权
- SaToken JWT认证
- 角色权限控制
- 接口资源权限控制

### 6.2 数据安全
- 密码加密存储
- SQL注入防护
- XSS防护
- 接口限流

### 6.3 隐私保护
- IP地址可配置显示
- 地理位置可配置显示
- 设备信息可配置显示

## 7. 性能优化

### 7.1 缓存策略
- 配置信息缓存（1小时）
- 热点文章缓存（30分钟）
- 用户信息缓存（2小时）

### 7.2 异步处理
- 邮件通知异步
- 日志记录异步（操作日志同步记录耗时）
- 统计任务定时执行

### 7.3 数据库优化
- 索引优化
- 分页查询
- 逻辑删除

## 8. 扩展性设计

### 8.1 存储扩展
- 文件存储支持本地和OSS切换
- 配置化切换

### 8.2 搜索扩展
- Elasticsearch可扩展
- 支持多索引

### 8.3 通知扩展
- 支持多种通知方式
- 可配置通知模板

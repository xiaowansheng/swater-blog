# 开发与本地启动

本文档面向第一次拉起项目或日常联调开发。

## 1. 环境要求

- JDK 21+
- Node.js 18+
- Docker / Docker Compose V2
- MySQL 8、Redis 7、RabbitMQ 3

## 2. 目录职责

- `blog-service`：Spring Boot 后端，默认端口 `8888`
- `blog-admin`：管理后台，Vite 开发端口 `3000`
- `blog-web`：博客前台，Next.js 开发端口 `3001`
- `blog-tools/yuque-sync`：语雀同步工具，开发端口 `3002`

## 3. 推荐启动方式

### 3.1 仅启动基础依赖

```bash
docker compose -f docker-compose.env.yml up -d mysql redis rabbitmq
```

适合本机运行 `blog-service`、`blog-admin`、`blog-web`。

### 3.2 启动基础依赖和后端容器

```bash
docker compose -f docker-compose.dev.yml up -d
```

适合只在本机调试前端，后端跑在容器里。

### 3.3 完整 Docker 本机访问（不改域名）

如果管理后台、博客前台、后端都跑在 Docker 里，本机没有域名也可以直接用 `localhost`。

`.env` 建议至少确认这些值：

```env
COMPOSE_PROFILES=http
SPRING_PROFILES_ACTIVE=docker
SERVER_PORT=8888
BLOG_ADMIN_PUBLISHED_PORT=3001
BLOG_WEB_PUBLISHED_PORT=3002

DB_HOST=localhost
DB_PORT=3306
DB_NAME=blog
DB_USER=blog
DB_PASSWORD=change_me_db_password
DB_ROOT_PASSWORD=change_me_root_password

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PUBLISHED_PORT=6379
REDIS_PASSWORD=
REDIS_DATABASE=0

RABBITMQ_HOST=localhost
RABBITMQ_PORT=5672
RABBITMQ_USER=admin
RABBITMQ_PASSWORD=change_me_rabbitmq_password
RABBITMQ_VHOST=/
```

启动：

```bash
docker compose -f docker-compose.yml -p swater-blog up -d --build
```

访问：

- 博客前台：`http://localhost`
- 管理后台：`http://localhost:3001`
- 后端健康检查：`http://localhost:8888/actuator/health`
- Swagger：`http://localhost:8888/swagger-ui.html`

说明：

- `http://localhost` 走外层 Nginx，前台的 `/api`、`/uploads` 会被代理到正确容器
- `http://localhost:3001` 直连管理后台容器，管理后台容器内置 Nginx 会代理 `/api`
- `BLOG_WEB_PUBLISHED_PORT` 只是 `blog-web` 容器直连端口，不建议作为前台主入口
- 当前根目录 `.env` 不会自动注入前端镜像构建变量；管理后台和前台应优先走 Nginx 暴露的 `/api`、`/uploads`

如果愿意使用本地域名，也可以在 hosts 中添加：

```txt
127.0.0.1 admin.local
127.0.0.1 web.local
```

然后访问 `http://admin.local` 和 `http://web.local`。

## 4. 本机联调步骤

### 4.1 准备环境变量

```bash
cp .env.example .env
```

常用变量：

- `DB_NAME` / `DB_USER` / `DB_PASSWORD`
- `REDIS_HOST` / `REDIS_PORT`
- `RABBITMQ_HOST` / `RABBITMQ_USER` / `RABBITMQ_PASSWORD`
- `SERVER_PORT`
- `MAIL_USERNAME` / `MAIL_PASSWORD`，仅在本机后端需要发信时配置

### 4.1.1 管理员初始化

首次空库启动时，后端会自动创建管理员账号。当前账号配置直接来自 Spring profile YAML：

说明：

- 直接运行本机 `dev` profile 时读取 [application-dev.yml](../../blog-service/src/main/resources/application-dev.yml)，默认 `admin/admin`
- 运行 `docker-compose.dev.yml` 或完整 Compose 时读取 [application-docker.yml](../../blog-service/src/main/resources/application-docker.yml)，默认 `admin/change_me_admin_password`
- 管理员用户已存在时不会重置密码
- 需要自定义初始化账号时，先修改对应 profile 的 `blog.bootstrap.admin.*`，再启动空库环境

### 4.2 启动后端

```bash
cd blog-service
set -a
source ../.env
set +a
./gradlew bootRun --args='--spring.profiles.active=dev'
```

开发环境实际读取：

- [application.yml](../../blog-service/src/main/resources/application.yml)
- [application-dev.yml](../../blog-service/src/main/resources/application-dev.yml)

说明：

- 根目录 `.env` 主要给 Docker Compose 使用
- 直接运行 `blog-service` 时，需要先把 `.env` 中的变量导入当前 shell 或 IDE 运行配置
- 通常需要导入 `DB_*`、`REDIS_*`、`RABBITMQ_*`；邮件、Token、存储、地图等变量按功能需要再补充

默认地址：

- API：`http://localhost:8888`
- Swagger：`http://localhost:8888/swagger-ui.html`
- 监控：`http://localhost:8081/actuator`

### 4.3 启动管理后台

```bash
cd blog-admin
npm install
npm run dev
```

默认地址：`http://localhost:3000`

可选本地覆盖：

- 复制 `blog-admin/.env.example` 为 `blog-admin/.env.local`
- 当前 Docker 构建不会从根目录 `.env` 注入管理后台变量；如需定制生产构建变量，需要先在 Compose/Dockerfile 中补充显式注入

### 4.4 启动博客前台

```bash
cd blog-web
npm install
npm run dev
```

默认地址：`http://localhost:3001`

可选本地覆盖：

- 复制 `blog-web/.env.example` 为 `blog-web/.env.local`
- 如果不额外配置，开发环境默认回落到 `http://localhost:8888`

说明：

- `blog-web` 的本地开发端口是 `3001`
- `blog-service` 开发环境中的 `revalidate.url` 已默认指向 `http://127.0.0.1:3001`

## 5. 辅助脚本

### Linux / macOS

- `./scripts/dev-setup.sh`：检查环境、安装依赖、启动基础服务
- `./scripts/start-dev.sh`：拉起基础服务并打开多个终端窗口

注意：

- `start-dev.sh` 依赖 `gnome-terminal`
- 如果你的桌面环境没有这个命令，建议手动按上面的步骤启动
- 仓库在 WSL 文件系统时，请在 WSL 终端里运行这些脚本和 `docker compose` 命令

### Windows

- `scripts\\dev-setup.bat`
- `scripts\\start-dev.bat`

注意：

- 如果当前目录是 `\\wsl.localhost\...`，不要用这两个 `.bat` 脚本启动 Compose
- 这种场景请切到 WSL 终端，在仓库目录里运行 `docker compose ...`

## 6. 常见问题

### 6.1 端口占用

需要检查这些端口：

- `80`：完整 Docker HTTP 模式的外层 Nginx，前台推荐入口
- `3000`：管理后台
- `3001`：博客前台
- `3002`：语雀同步工具
- `8888`：后端 API
- `8081`：监控端点
- `3306` / `6379` / `5672` / `15672`：基础依赖

如果 `80` 被占用，完整 Docker 模式下 `http://localhost` 会启动失败。处理方式：

- 释放本机 `80` 端口后重新启动 Compose
- 或者修改 [docker-compose.yml](../../docker-compose.yml) 中 `nginx` 服务的端口映射，例如把 `"80:80"` 改成 `"8080:80"`，之后前台访问 `http://localhost:8080`

### 6.2 RabbitMQ 登录信息

开发编排默认值来自 `.env` / `.env.example`：

- 用户名：`admin`
- 密码：查看 `RABBITMQ_PASSWORD`

管理地址：`http://localhost:15672`

### 6.3 Docker 与本机运行差异

- `docker-compose.env.yml` 只启动依赖服务
- `docker-compose.dev.yml` 会额外启动 `blog-service`，且该容器当前使用 `docker` profile
- 完整前后端容器化使用根目录 [docker-compose.yml](../../docker-compose.yml)
